# ☁️ Cloud Computing – Complete Guide (AWS)

This repository contains fundamental concepts of Cloud Computing with a focus on AWS, designed for beginners and DevOps learners.

---

## 1. What is Cloud Computing

Cloud computing is the **on-demand delivery of computing resources** such as servers, storage, databases, networking, and software over the internet with pay-as-you-go pricing.

### Features:
- No upfront cost
- Scalable resources
- High availability
- Global access

---

## 2. Evolution of Cloud Computing

- **Traditional IT** → Physical servers, manual management  
- **Virtualization** → Multiple VMs on one server  
- **Cloud (2006+)** → On-demand infrastructure  
- **Cloud Native** → Containers, microservices, DevOps  
- **Serverless** → No server management  

---

## 3. Traditional IT vs Cloud

| Feature | Traditional IT | Cloud |
|--------|--------------|------|
| Cost | High upfront | Pay-as-you-go |
| Scalability | Manual | Automatic |
| Deployment | Slow | Fast |
| Maintenance | Manual | Managed |
| Availability | Limited | High |

---

## 4. Business Challenges Solved by Cloud

- High infrastructure cost → Reduced cost  
- Scaling issues → Auto scaling  
- Downtime → High availability  
- Slow deployment → Faster delivery  
- Hardware dependency → Virtual infrastructure  

---

## 5. Cloud Deployment Models

### Public Cloud
- Shared infrastructure
- Accessible over internet

### Private Cloud
- Dedicated infrastructure
- More control & security

### Hybrid Cloud
- Combination of public + private cloud

---

## 6. Cloud Service Models

### IaaS
- Infrastructure provided
- User manages OS & apps

### PaaS
- Platform provided
- Focus on development

### SaaS
- Ready-to-use software
- No management required

---

## 7. Core Cloud Principles

- **Elasticity** → Auto scale resources  
- **Scalability** → Handle increasing load  
- **High Availability** → Minimal downtime  
- **Fault Tolerance** → System works even on failure  

---

## 8. Shared Responsibility Model

### Cloud Provider:
- Hardware
- Networking
- Data centers
- Physical security

### User:
- Applications
- Data
- Access control
- Configuration

---

## 10. Why AWS

## 11. AWS Global Infrastructure

### Regions
- Geographical locations (e.g., Mumbai, US-East)

### Availability Zones
- Multiple isolated data centers in a region

### Benefits
- High availability  
- Fault isolation  
- Disaster recovery
